`open-slide build` produces a plain static site under `dist/`. The output is
zero-runtime — everything renders on the client, but no server is needed.

## Static HTML

```npm
npm run build
npm run preview     # serves dist/ for sanity-checking
```

`dist/` is portable. Drop it on Vercel, Cloudflare Pages, Netlify, Zeabur,
GitHub Pages — anything that hosts files.

## PDF

The dev server can print the active deck to PDF via the browser's native
print pipeline. From the toolbar pick **Export → PDF**, or open the deck at
`?print=1` and use Cmd/Ctrl-P. The runtime sets paper size to 1920×1080
landscape so each slide is exactly one page.

## PPTX

From the toolbar, choose one of three presentation exports:

- **Editable PPTX** converts text, images, solid fills, borders, and basic shapes
  into native presentation objects. Complex CSS, gradients, SVG, canvas, video,
  and other browser-only effects are kept as raster fallback regions.
- **Image PPTX** captures each complete page as an image. It is the
  highest-fidelity option, but its contents are not editable.
- **Google Slides PPTX** keeps text, images, solid fills, borders, and basic
  shapes editable while using Google Slides-compatible fonts and conservative
  text settings. Unsupported CSS regions remain images.

The editable exporter reports how many complex regions it retained as images.
Animations and stepped reveals are exported in their final settled state.
Speaker notes exported from the slide module are included.

## Use with Google Slides

Choose **Export for Google Slides**, upload the resulting
`*-google-slides.pptx` file to Google Drive, and select **Open with → Google
Slides**. Supported elements remain editable. Fonts available in Google Slides
are preserved; unsupported fonts fall back to Arial to reduce substitution and
text reflow. No Google credentials or cloud integration are required by
open-slide.

Complex CSS, SVG, video, canvas, gradients, and browser-only effects remain
raster fallback regions. Use **Editable PPTX** instead when PowerPoint is the
destination and preserving the original font and spacing settings matters more.

### Preserve complex visualizations exactly

Add `data-pptx-raster` to the outermost container of a diagram, chart, icon
composition, or other complex visual that must remain visually exact:

```tsx
<div data-pptx-raster style={{ position: 'relative' }}>
  <Diagram />
  <Icons />
  <Decorations />
</div>
```

The exporter captures that complete container as one image in both editable
PowerPoint and Google Slides exports. Content outside the marked container
remains editable. Place headings, captions, and explanatory text outside the
marked region when they need to remain editable.

Use one marker on the common outer container rather than marking each child.
This is especially important for overlapping elements, SVG compositions,
gradients, masks, shadows, transforms, and browser-rendered charts. SVG,
canvas, and video elements are also detected automatically.

## What gets shipped

The build pipeline only includes:

- The decks under `slides/`.
- The assets each deck imports.
- The runtime — viewer, scaler, present mode (configurable; see below).

You can trim the surface area for a public share by toggling fields in
`open-slide.config.ts`:

```ts
import type { OpenSlideConfig } from '@open-slide/core';

const config: OpenSlideConfig = {
  build: {
    showSlideBrowser: false,    // hide the deck index
    showSlideUi: false,         // hide on-canvas UI (toolbar etc.)
    allowHtmlDownload: false,   // hide the "download HTML" button
  },
};

export default config;
```
